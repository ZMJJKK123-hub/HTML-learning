# 第 2 步：页面结构详解 - 构建一个真实网页

## 📌 文件说明
这个文件展示了一个**真实网页的结构**。你会看到：
- 更多的 `<meta>` 标签（用于配置）
- 语义化的 HTML 标签（`<header>`, `<nav>`, `<main>` 等）
- 一个完整的个人网站框架

---

## 🎯 核心概念：语义化标签

什么是"语义化"？就是**使用有含义的标签来描述内容**。

### 为什么要使用语义化标签？

**不好的做法（没有语义）：**
```html
<div>我的网站</div>
<div><a href="#">关于</a></div>
<div>我是开发者</div>
```
浏览器和搜索引擎看到的只是一堆 `<div>`，不知道这些是页面结构的什么部分。

**好的做法（有语义）：**
```html
<header>
    <h1>我的网站</h1>
</header>
<nav>
    <a href="#">关于</a>
</nav>
<main>
    <p>我是开发者</p>
</main>
```
浏览器和搜索引擎立即知道这是页面的头部、导航和主要内容。

---

## 🔍 Head 部分详解（新增内容）

### 第 1-2 行：基础配置
```html
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

**`charset="UTF-8"`**
- 前面已经讲过了，用来支持中文

**`viewport` 是什么？**
- 视口（viewport）是用户看到网页的区域
- 这行代码表示：
  - `width=device-width` - 网页宽度等于设备宽度（手机、平板、电脑）
  - `initial-scale=1.0` - 初始缩放比例为 100%
- **为什么重要**：这使网页能够**在手机上正确显示**（响应式设计）

### 第 3-5 行：页面元数据
```html
<meta name="description" content="我的个人网站，展示我的信息和项目">
<meta name="author" content="张三">
<title>张三 - 前端开发工程师</title>
```

**`name="description"`**
- 页面的描述信息
- 搜索引擎（Google、百度）会用这个描述来显示你的网页
- 用户在搜索结果中会看到这段描述

**`name="author"`**
- 页面的作者
- 用来标记谁创建了这个网页

## 🏗️ Body 部分详解（语义化标签）

### `<header>` - 页头
```html
<header>
    <h1>张三的个人网站</h1>
    <p>欢迎来到我的网站！</p>
</header>
```

**作用**：
- 包含页面的头部信息
- 通常包含：网站名称、标题、欢迎语
- 放在 `<body>` 的最开始

**真实网站中的应用**：
```
┌─────────────────────────────────┐
│  LOGO        张三的网站          │  ← 这就是 <header>
│              欢迎来到我的网站     │
└─────────────────────────────────┘
```

---

### `<nav>` - 导航菜单
```html
<nav>
    <a href="#about">关于我</a>
    <a href="#skills">技能</a>
    <a href="#projects">项目</a>
    <a href="#contact">联系方式</a>
</nav>
```

**作用**：
- 包含导航链接
- 用户可以通过这些链接在网页的不同部分之间跳转

**什么是 `<a>` 标签？**
- 锚点（anchor），用来创建链接
- `href="#about"` 表示点击时跳转到 id="about" 的元素
- `#` 表示这是同一页面内的跳转（不是外部链接）

**如何工作？**
```
用户点击 "关于我"
    ↓
浏览器找到 id="about" 的位置
    ↓
页面滚动到那个位置
```

---

### `<main>` - 主要内容
```html
<main>
    <section id="about">
        ...
    </section>
    <section id="skills">
        ...
    </section>
    <section id="projects">
        ...
    </section>
</main>
```

**作用**：
- 包含页面的**主要内容**
- 通常是最重要的部分
- 每个页面应该只有一个 `<main>`

---

### `<section>` - 内容分块
```html
<section id="about">
    <h2>关于我</h2>
    <p>我是一名前端开发工程师...</p>
</section>
```

**作用**：
- 将相关内容分组在一起
- 每个 `<section>` 通常有自己的标题（`<h2>`）

**`id="about"` 有什么用？**
- 给这个部分起一个唯一的"名字"
- 用于导航菜单的链接跳转
- 也可以用于 CSS 样式和 JavaScript 操作

**真实结构：**
```
<main>
    ├─ <section id="about">       关于我部分
    ├─ <section id="skills">      技能部分
    └─ <section id="projects">    项目部分
```

---

### `<article>` - 文章
```html
<article>
    <h3>项目 1：电商网站</h3>
    <p>用 React 和 Node.js 构建的全栈电商平台</p>
</article>
```

**`<article>` 和 `<section>` 的区别？**

| 标签 | 用途 | 例子 |
|------|------|------|
| `<section>` | 将相关内容分组 | 页面的某个部分（关于我、技能）|
| `<article>` | 独立的、可重用的内容 | 博客文章、新闻、评论 |

简单说：`<article>` 是可以独立分享的内容（比如发到另一个网站也能理解）

---

### `<aside>` - 侧边栏
```html
<aside>
    <h3>最新发布</h3>
    <ul>
        <li>新文章：HTML 基础入门</li>
        <li>更新：项目 1 完成</li>
    </ul>
</aside>
```

**作用**：
- 放置**辅助信息**（不是主要内容）
- 通常显示在页面的侧边
- 例如：最新发布、热门文章、广告等

---

### `<footer>` - 页脚
```html
<footer>
    <p>&copy; 2024 张三. 版权所有。</p>
    <p>联系邮箱：zhangsan@example.com</p>
</footer>
```

**作用**：
- 页面的底部信息
- 通常包含：版权、联系方式、法律条款、网站地图等

**什么是 `&copy;`？**
- HTML 实体，表示 © 符号（版权符号）
- 浏览器会自动将 `&copy;` 转换成 ©

---

```
<!DOCTYPE html>

    <body>

```

## 🔗 链接的工作方式

### 外部链接（跳转到其他网站）
```html
<a href="https://www.baidu.com">访问百度</a>
```
- `href` 是完整的 URL
- 点击后会打开新的网站

### 内部链接（同一页面跳转）
```html
<a href="#about">关于我</a>    <!-- 导航菜单 -->
<section id="about">           <!-- 目标位置 -->
    <h2>关于我</h2>
</section>
```
- `href="#about"` 中的 `#` 表示页面内跳转
- `id="about"` 是跳转的目标
- 点击链接时，页面会自动滚动到对应的位置

### 在新标签页打开
```html
<a href="https://www.google.com" target="_blank">Google</a>
```
- `target="_blank"` 表示在新标签页打开链接

## 💡 HTML 注释

```html
<!-- 这是一个注释，不会在网页上显示 -->
```

**什么是注释？**
- 用来给代码添加说明，只有开发者能看到
- 浏览器会忽略注释，不会显示在网页上
- 格式：`<!-- 注释内容 -->`

**为什么要写注释？**
- 帮助自己和其他开发者理解代码的用途
- 标记重要的部分
- 暂时禁用某些代码

---

## 🎯 重要的 HTML 实体

有些符号在 HTML 中有特殊含义，所以需要用特殊的代码表示：

| 显示 | 代码 | 说明 |
|------|------|------|
| © | `&copy;` | 版权符号 |
| ® | `&reg;` | 注册商标 |
| ™ | `&trade;` | 商标 |
| & | `&amp;` | 和号 |
| < | `&lt;` | 小于号 |
| > | `&gt;` | 大于号 |
| " | `&quot;` | 双引号 |
| ' | `&apos;` | 单引号 |
| 空格 | `&nbsp;` | 不换行空格 |

这个文件中用到了 `<ul>` 和 `<li>`：

