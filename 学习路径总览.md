# 学习路径总览

本页汇总 HTML/CSS/JS 1–20 步与 15 篇补充专题的推荐顺序与学习建议。

## 路径建议（从零到部署）
1) HTML 基础（1–7）→ 打牢语义与结构
2) CSS 进阶（8–14）→ 布局、动效与主题
3) JS 主线（15–20）→ 语法、DOM、异步、组件、性能、综合项目
4) 补充专题（01–06）→ 网络性能、安全、可访问性、工程化、架构、部署与 PWA
5) 扩展专题（07–15）→ 框架与 SSR、状态库、后端协作、测试、设计系统、CI/CD、可视化、安全进阶、移动与跨端

## 不同目标的学习路径
- 产品落地（快速上线）
  - 1–14 → 15–18 → 补充 01/02/06 → 20 → 补充 12（CI/CD）
- 工程与质量（规范/测试/部署）
  - 1–20 → 补充 04（工程化与测试）→ 12（CI/CD）→ 02（安全）→ 06（PWA）
- 框架上手（React/Vue/Svelte）
  - 1–14 → 15–18 → 补充 07（框架与SSR）→ 08（数据获取与状态库）→ 11（设计系统）

## 每阶段的目标
- HTML 阶段：掌握语义与可访问性基本原则；表单与表格的结构与提示。
- CSS 阶段：能实现响应式、Grid/Flex 布局与基础动效；使用 CSS 变量管理主题。
- JS 阶段：熟悉模块化、DOM/事件、异步与状态；具备组件化思维与性能优化意识。
- 补充阶段：把项目做实（性能、安全、工程化、部署），形成端到端闭环。

## 学习建议
- 小步快跑：每完成 1 步，做至少 1 个基础任务 + 1 个挑战任务。
- 记录与复盘：用自查清单（见“提交作业与自查指南”）记录收获与问题。
- 实战驱动：尽量把补充专题的练习落到“20 综合项目”中，形成整站能力。
