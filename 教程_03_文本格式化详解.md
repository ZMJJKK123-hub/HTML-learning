# 第 3 步：文本格式化详解 - 让文字更有表现力

## 📌 文件说明
这个文件展示了 HTML 中所有常见的**文本格式化标签**。每个标签都有特定的用途，用来让文字表现不同的含义和样式。

---

## 🎯 核心概念：语义化 vs 样式

在学习文本格式化时，有一个重要的原则：

**区分语义和样式：**

| 标签 | 类型 | 含义 | 用途 |
|------|------|------|------|
| `<strong>` | 语义 | 内容很重要 | 标记重要信息 |
| `<b>` | 样式 | 仅仅是加粗 | 样式装饰 |
| `<em>` | 语义 | 内容需要强调 | 强调某部分 |
| `<i>` | 样式 | 仅仅是斜体 | 书名、外文等 |

**为什么这很重要？**
- 搜索引擎通过语义标签理解内容的重要性
- 屏幕阅读器（给盲人用的）会用不同的语调读出被标记的文字
- 使用正确的语义标签使网页更易被理解

---

## 🔍 详细讲解每个标签

### 第 1 类：标题标签（h1 - h6）

```html
<h1>一级标题</h1>
<h2>二级标题</h2>
<h3>三级标题</h3>
... 以此类推到 h6
```

**为什么有 6 个等级？**
- `<h1>` - 页面的主标题，最大最重要（**每页只有一个**）
- `<h2>` - 主要内容的标题（通常有多个）
- `<h3>` - 子内容的标题
- ...
- `<h6>` - 最小的标题

**真实网站结构的例子：**
```
<h1>我的个人博客</h1>          ← 网站主标题

<h2>第一篇文章：HTML基础</h2>   ← 文章标题
<h3>什么是HTML？</h3>           ← 文章的章节
<h3>HTML标签</h3>

<h2>第二篇文章：CSS入门</h2>    ← 另一篇文章
<h3>什么是CSS？</h3>
```

**规则：**
- ✅ 标题应该按照逻辑顺序使用（h1 → h2 → h3）
- ❌ 不要跳过等级（比如 h1 → h3，中间没有 h2）
- ❌ 不要为了让标题变小而使用低等级标题，应该用 CSS

### 第 2 类：强调类标签

#### `<strong>` - 强烈强调（内容很重要）
```html
这是 <strong>很重要的信息</strong>。
```
**使用场景：**
- 警告信息：`<strong>警告：</strong> 不要删除这个文件！`
- 重要指示：`<strong>必须</strong> 在之前完成这个步骤`
- 关键词：`<strong>响应式设计</strong> 是现代网页开发的核心`

**浏览器显示：** 加粗
**屏幕阅读器：** 用不同的语调读出，表示很重要

#### `<b>` - 加粗（仅样式，无语义）
```html
<b>这是加粗文字</b>
```
**使用场景：**
- 产品摘要：`<b>摘要：</b> 这个产品有...`
- 导语：`<b>开篇语：</b> 本文将讨论...`
- 当你只是想让文字看起来加粗，没有特殊含义

**区别：**
```html
✅ 好的做法：
<strong>重要</strong>：请立即更新系统。

❌ 不好的做法：
<b>重要</b>：请立即更新系统。
（因为屏幕阅读器不知道这是重要信息）
```

---

#### `<em>` - 强调（内容需要重点关注）
```html
这对我 <em>非常重要</em>。
```
**使用场景：**
- 强调观点：`我<em>真的</em>喜欢这部电影`
- 对比：`这是<em>好的</em>选择，不是那个`
- 语气强调：`请<em>一定要</em>记住这一点`

**浏览器显示：** 斜体
**屏幕阅读器：** 改变语调来强调

#### `<i>` - 斜体（仅样式，无语义）
```html
我喜欢看 <i>三体</i> 这部作品。
```
**使用场景：**
- 书名：`<i>活着</i>`
- 电影名：`<i>肖申克的救赎</i>`
- 外文词：`<i>café</i>`（法文的咖啡馆）
- 技术术语：`<i>Java</i> 编程语言`
- 人名（某些风格）：`由 <i>史蒂芬·霍金</i> 著述`

**区别：**
```html
<em>这很重要</em>        - 重点强调
<i>这是外文词</i>        - 仅是样式
<i>爱因斯坦</i>          - 外文人名
```

### 第 3 类：删除和标记

#### `<del>` - 删除的文本
```html
原价：<del>¥999</del> 现价：¥599
```
**用途：** 显示已删除的内容，表示价格变化、错误更正等

**常见场景：**
- 价格折扣：`<del>¥999</del> → ¥599`
- 编辑修订：`这个版本<del>有bug</del>已修复`
- 过时信息：`<del>2023年的规则</del> 已更新为2024年规则`

#### `<s>` - 不再准确的文本
```html
状态：<s>进行中</s> 已完成
```
**使用场景：**
- 产品状态变化：`<s>库存：100件</s> 已售罄`
- 计划更改：`<s>会议时间：下午3点</s> 改为晚上6点`

**`<del>` 和 `<s>` 的区别：**
- `<del>` 表示内容被**删除**了（更正）
- `<s>` 表示内容**不再准确**（但仍然存在）

```html
✅ 使用 <del>：
原来是错的：2+2=5，应该是 2+2=4
正确的说法：原来是<del>2+2=5</del> 2+2=4

✅ 使用 <s>：
计划改变了：<s>明天开会</s> 改到后天
```

#### `<mark>` - 高亮标记
```html
搜索关键词 "HTML"，结果：
<mark>HTML</mark> 是网页的基础。用 <mark>HTML</mark> 可以...
```
**使用场景：**
- 搜索结果高亮
- 强调相关部分
- 标记重点

---

### 第 4 类：上标和下标

#### `<sup>` - 上标
```html
数学：E=mc<sup>2</sup>
```
**使用场景：**
- 数学公式：`x<sup>2</sup> + y<sup>2</sup> = z<sup>2</sup>`
- 注脚标记：`Word<sup>1</sup>`
- 商标：`Smart<sup>®</sup>`
- 版权：`<sup>©</sup>2024`

#### `<sub>` - 下标
```html
化学：H<sub>2</sub>O
```
**使用场景：**
- 化学公式：`H<sub>2</sub>SO<sub>4</sub>`（硫酸）
- 数学：`a<sub>1</sub>, a<sub>2</sub>, a<sub>n</sub>`
- 脚注编号：`见参考资料<sub>[1]</sub>`

---

### 第 5 类：代码和技术类

#### `<code>` - 代码
```html
函数 <code>console.log()</code> 用来输出信息。
```
**使用场景：**
- 提及函数：`使用 <code>Math.random()</code> 生成随机数`
- 变量名：`变量 <code>counter</code> 用来计数`
- HTML标签：`<code>&lt;div&gt;</code> 标签用来分组`

**注意：** `<code>` 用于**短的代码片段**，整个代码块应该用 `<pre><code>` 包裹

#### `<kbd>` - 键盘输入
```html
按 <kbd>Ctrl</kbd> + <kbd>S</kbd> 保存
```
**使用场景：**
- 快捷键：`<kbd>Ctrl</kbd> + <kbd>C</kbd>` 复制
- 按钮：`点击 <kbd>OK</kbd> 按钮`
- 菜单路径：`<kbd>文件</kbd> → <kbd>另存为</kbd>`

#### `<samp>` - 程序输出
```html
命令输出：<samp>Hello, World!</samp>
```
**使用场景：**
- 展示程序的输出结果
- 终端命令的输出
- 软件的返回信息

#### `<pre>` - 预格式化文本
```html
<pre>
function add(a, b) {
    return a + b;
}
</pre>
```
**特点：**
- **保留所有空格和换行**（不像普通文本会被压缩）
- 通常用 `monospace` 字体（等宽字体）显示
- 完美用于显示代码块、ASCII 艺术、日志等

**为什么需要 `<pre>`？**
```
不用 <pre>：
function add(a, b) {
    return a + b;
}
浏览器会显示成：function add(a, b) { return a + b; }
（所有的缩进和换行都被忽略了）

用 <pre>：
<pre>
function add(a, b) {
    return a + b;
}
</pre>
浏览器保留所有缩进和换行
```

### 第 6 类：引用

#### `<q>` - 短引用
```html
有句名言：<q>生活就像骑自行车，为了保持平衡，你必须不断前进。</q>
```
**特点：**
- 用于短的引用（一句话）
- 浏览器会自动添加引号（双引号）
- 表示这是他人的观点或词汇

**使用场景：**
- 引用名言：`<q>知识就是力量</q> — 培根`
- 引用他人的话：`她说 <q>我很开心</q>`

#### `<blockquote>` - 长引用
```html
<blockquote>
    <p>这是一个较长的引用段落。</p>
    <p>它可以包含多个段落。</p>
    <p>— 引用出处</p>
</blockquote>
```
**特点：**
- 用于较长的引用（多个段落）
- 通常会自动缩进（由浏览器或CSS控制）
- 可以包含多个段落

**属性：**
- `cite` - 引用来源的 URL
```html
<blockquote cite="https://www.example.com/article">
    这是引用的内容。
</blockquote>
```

**使用场景：**
- 引用文章内容
- 引用书籍段落
- 客户评价
- 推荐语

---

### 第 7 类：其他重要标签

#### `<abbr>` - 缩写词
```html
<abbr title="HyperText Markup Language">HTML</abbr>
```
**使用场景：**
- `<abbr title="Cascading Style Sheets">CSS</abbr>`
- `<abbr title="Application Programming Interface">API</abbr>`
- `<abbr title="JavaScript Object Notation">JSON</abbr>`

**实际效果：** 当用户鼠标悬停在 HTML 上时，会看到完整的说明文本。

#### `<dfn>` - 定义
```html
<dfn>响应式设计</dfn> 是指网页能够适应不同的屏幕尺寸。
```
**使用场景：**
- 首次定义一个术语
- 在教学中解释概念
- 创建词汇表

#### `<address>` - 联系方式
```html
<address>
    公司地址：北京市朝阳区<br>
    电话：010-1234-5678<br>
    邮箱：contact@example.com
</address>
```
**使用场景：**
- 网站联系方式
- 公司信息
- 页脚中的地址信息

#### `<cite>` - 作品标题
```html
我推荐阅读 <cite>活着</cite> 这部作品。
```
**使用场景：**
- 书籍标题：`<cite>活着</cite>`
- 电影名：`<cite>肖申克的救赎</cite>`
- 音乐标题：`<cite>夜曲</cite>`

---

### 第 8 类：换行和分割

#### `<br>` - 换行
```html
第一行 <br>
第二行 <br>
第三行
```
**注意：** 避免过度使用 `<br>`！应该用 `<p>` 标签分组段落。

**好的用法：**
- 地址：`北京市 <br> 朝阳区`
- 诗歌：每行一个 `<br>`
- 列表项：每项一个 `<br>`

**不好的用法：**
```html
❌ 不要这样做：
第一段文字 <br>
<br>
<br>
第二段文字

✅ 应该这样做：
<p>第一段文字</p>
<p>第二段文字</p>
```

#### `<hr>` - 水平分割线
```html
第一个话题
<hr>
第二个话题
```
**用途：** 在两个主要内容块之间添加分割线，表示主题变化。

**使用场景：**
- 分离不同的主题
- 分离不同的章节
- 视觉上的停顿

## 📌 最重要的规则

1. **优先使用语义化标签** - 它们帮助搜索引擎和辅助工具理解内容
2. **不要过度装饰** - 不是每个词都需要格式化
3. **保持一致** - 在整个网站中使用一致的标记方式
4. **使用 CSS 而不是 HTML 来控制样式** - HTML 是结构，CSS 是样式
