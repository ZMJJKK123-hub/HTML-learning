# 提交作业与自查指南

适用：完成分步作业（01–20）与补充专题练习时的记录与自评方式。

## 提交格式（示例）
- 标题：第 X 步作业 + 练习编号（如：第 11 步作业）
- 内容：
  - 完成项：列出基础/挑战任务的实现点与文件路径
  - 截图或简述：关键界面或交互说明
  - 反思：遇到的问题、下次改进计划

## 自查清单（通用）
- 语义与结构：标签使用是否合理、标题层级是否正确
- 可访问性：键盘可达、aria 属性、颜色对比达标
- 交互与状态：加载/成功/错误三态是否可见且一致
- 安全与健壮性：输入校验、避免直接 innerHTML 注入、错误处理与重试
- 性能与工程化：按需加载、缓存策略、指纹与压缩
- 代码质量：命名清晰、模块化、避免重复、Lint 通过

## 记录方式建议
- 在每步作业的文件末尾添加“自查小结”段落
- 建立一个自己的“学习日志.md”记录每日进度与收获
- 使用 Git 提交：每个作业单独 commit，编写有意义的提交信息
